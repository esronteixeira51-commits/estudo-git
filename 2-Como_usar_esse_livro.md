# Guia do Leitor

## Como usar este livro

> *Este livro foi pensado para servir dois tipos de leitores ao mesmo tempo.*

---

## 01 — Para quem vai estudar do zero

Leia na ordem. Cada parte foi construída sobre a anterior — pular etapas vai criar lacunas que aparecem na hora errada.

| Parte | Conteúdo |
|---|---|
| **Parte 0** | Configuração inicial — não pule! |
| **Parte 1** | Fundamentos |
| **Parte 2** | Colaboração |
| **Parte 3** | Nível Profissional |
| **Parte 4** | Automação e CI/CD |

Em cada capítulo, faça os exercícios práticos. Não apenas leia os comandos — **execute-os no seu computador**. O Git só entra na cabeça quando as mãos trabalham.

Recomendo estudar um ou dois capítulos por dia, praticando bastante. Se você dedicar **1h a 1h30 por dia**, em cerca de **3 a 4 semanas** terá um nível muito sólido.

---

Vá direto para a **Parte 5 — Referência Rápida**:

## 03 — Dicas práticas de estudo

- Crie uma pasta chamada `treino-git` e use ela para todos os exercícios.
- Mantenha o terminal do VS Code sempre aberto enquanto lê.
- Quando encontrar um comando novo, teste variações dele. Brinque.
- Não tenha medo de "quebrar" os repositórios de treino — é exatamente para isso que eles existem.
- Depois de cada capítulo, tente explicar com suas próprias palavras o que você acabou de aprender (**técnica Feynman**). Se conseguir explicar para alguém, você realmente entendeu.

---

> **Lembre-se:** este livro não é para ser lido passivamente. É para ser usado, rabiscado, testado e consultado muitas vezes.

---

No final, o objetivo não é você saber todos os comandos de cor. O objetivo é você **entender o que o Git está fazendo** e conseguir resolver problemas sozinho — mesmo aqueles que não estão no livro.

*Agora chega de introdução.*

Abra o terminal, crie a pasta `treino-git` e vamos começar.
