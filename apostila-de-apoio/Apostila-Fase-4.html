1. Introdução à Fase 4

Profissionais que dominam Git + GitHub Actions são muito mais valorizados no mercado.

2. O que é GitHub Actions?

GitHub Actions é uma plataforma de automação integrada ao GitHub que permite criar workflows (fluxos de trabalho) em arquivos YAML.

Principais usos:
  • Executar testes automaticamente
  • Verificar qualidade de código (lint)
  • Buildar projetos
  • Fazer deploy automático
  • Notificações e relatórios

3. Estrutura Básica de um Workflow

Os workflows ficam na pasta .github/workflows/

Exemplo de arquivo: ci.yml

name: CI Pipeline

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test

4. Triggers (Eventos que disparam o workflow)

TriggerDescrição
pushQuando código é enviado para o repositório
pull_requestQuando um PR é aberto, atualizado ou sincronizado
workflow_dispatchExecução manual (botão no GitHub)
scheduleExecução agendada (cron)
releaseQuando uma release é criada

5. Jobs e Steps

  • Job: Um conjunto de passos que roda em um runner (máquina virtual)
  • Step: Cada ação individual dentro de um job

Exemplo com múltiplos jobs:

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm run lint

  test:
    runs-on: ubuntu-latest
    needs: lint
    steps:
      - uses: actions/checkout@v4
      - run: npm test

  deploy:
    runs-on: ubuntu-latest
    needs: test
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      - run: echo "Deploying to production..."

6. Exemplos Práticos de Workflows

1. Node.js + Testes

name: Node CI

on:
  push:
    branches: [ main ]
  pull_request:

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [18.x, 20.x]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
      - run: npm ci
      - run: npm run build --if-present
      - run: npm test

2. Deploy para Vercel / Netlify (simplificado)

- name: Deploy to Vercel
  uses: amondnet/vercel-action@v25
  with:
    vercel-token: ${{ secrets.VERCEL_TOKEN }}
    vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
    vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}

7. Recursos Avançados

  • Secrets → Armazenar chaves API, tokens e senhas com segurança
  • Environment Variables → Variáveis de ambiente
  • Matrix Strategy → Testar em múltiplas versões de Node, Python, etc.
  • Caching → Acelerar workflows (ex: node_modules)
  • Artifacts → Salvar arquivos gerados (builds, relatórios)
  • Reusable Workflows → Workflows reutilizáveis entre repositórios

8. Segurança e Boas Práticas

  • Use Secrets em vez de colocar credenciais no código
  • Ative Dependabot para atualizações de segurança
  • Ative Code Scanning e Secret Scanning
  • Configure Branch Protection Rules + Required Status Checks
  • Use pull_request para rodar workflows em PRs
  • Evite rodar workflows de forks sem permissão
Conventional Commits + GitHub Actions
Muitos times usam ações que geram automaticamente CHANGELOG e Releases baseadas nos commits.

9. Outras Ferramentas Avançadas

  • Git LFS → Gerenciar arquivos grandes (imagens, datasets)
  • Git Submodules → Repositórios dentro de repositórios
  • Monorepos → Gerenciar múltiplos projetos em um único repositório
  • GitHub API → Automatizar via scripts
  • Webhooks → Integrar com outras ferramentas

10. Exercícios Finais - Torne-se Profissional

  1. Crie um repositório e adicione um workflow que rode testes automaticamente em toda push e PR
  2. Configure um workflow com matrix strategy para testar em duas versões diferentes da linguagem
  3. Crie um workflow que faça deploy automático para GitHub Pages quando fizer merge na main
  4. Adicione Secrets no repositório e use em um workflow
  5. Ative Dependabot e configure atualizações automáticas
  6. Configure Branch Protection Rules com Required Status Checks (exija que o CI passe)
  7. Crie um workflow completo para um projeto real seu (frontend, backend ou fullstack)
  8. Contribua com um PR em um projeto open-source que use GitHub Actions
