1. Introdução ao Git e GitHub

Git é um sistema de controle de versão distribuído, criado por Linus Torvalds. Ele permite rastrear todas as alterações no seu código ao longo do tempo.

GitHub é uma plataforma online que hospeda repositórios Git, facilita a colaboração em equipe e oferece ferramentas extras (Pull Requests, Issues, Actions, etc.).

Por que aprender Git?
• Trabalhar em equipe sem bagunçar o código
• Voltar para versões anteriores facilmente
• Mostrar seu portfólio de forma profissional
• É exigência básica em quase todas as vagas de programação

2. Instalação do Git

Windows

Baixe em: git-scm.com/download/win

Mac

brew install git

ou baixe o instalador oficial.

Linux (Ubuntu/Debian)

sudo apt update && sudo apt install git

Após instalar, verifique a versão:

git --version

3. Configuração Inicial (obrigatório!)

Configure seu nome e e-mail. Esses dados aparecem em todos os commits.

git config --global user.name "Seu Nome Completo"
git config --global user.email "seuemail@exemplo.com"

Verifique as configurações:

git config --global user.name
git config --global user.email
git config --list
Use o mesmo e-mail cadastrado na sua conta do GitHub para que os commits fiquem vinculados corretamente.

4. Conceitos Básicos

ConceitoDescrição
Repositório (Repository)Pasta que contém todo o histórico do projeto
CommitUm "ponto no tempo" que salva as alterações
Branch (ramo)Linha de desenvolvimento (por padrão: main ou master)
RemoteRepositório remoto (ex: no GitHub)
.gitignoreArquivo que diz ao Git quais arquivos/pastas ignorar

5. Comandos Essenciais da Fase 1

Criar um repositório novo

mkdir meu-projeto
cd meu-projeto
git init

Clonar um repositório existente (do GitHub)

git clone https://github.com/usuario/nome-do-repo.git

Ver o status

git status

Adicionar arquivos

git add nome-do-arquivo.txt     # um arquivo
git add .                      # todos os arquivos

Fazer commit

git commit -m "Mensagem clara do que foi feito"

Ver histórico

git log
git log --oneline --graph

Diferenças

git diff                  # alterações não adicionadas
git diff --staged          # alterações já adicionadas

6. Ciclo de Vida dos Arquivos

Working Directory → Staging Area → Commit → Repositório

# Fluxo mais usado:
git status
git add .
git commit -m "feat: adiciona página inicial"
git status

7. Conectando com o GitHub

Passo a passo:

  1. Crie um repositório novo no site do GitHub (não marque "Add a README file")
  2. No terminal, dentro do seu projeto:
git remote add origin https://github.com/SEU-USUARIO/nome-do-repo.git
git branch -M main
git push -u origin main
Primeira vez no GitHub?
Crie uma conta gratuita em github.com e faça seu primeiro "Hello World" no GitHub.

8. Boas Práticas de Commit

Use o padrão Conventional Commits (recomendado):

  • feat: nova funcionalidade
  • fix: correção de bug
  • docs: alteração na documentação
  • chore: manutenção (atualizar dependências, etc.)
  • refactor: refatoração de código

Exemplos bons:

feat: adiciona formulário de contato
fix: corrige erro de validação no login
chore: atualiza dependências do projeto

Arquivo .gitignore

Crie um arquivo chamado .gitignore na raiz do projeto:

# Exemplo básico
node_modules/
*.log
.env
.DS_Store
dist/
build/

9. Exercícios Práticos (Faça todos!)

  1. Instale o Git e configure seu nome e e-mail
  2. Crie uma pasta chamada meu-primeiro-repo, inicialize o Git e faça 3 commits com arquivos diferentes
  3. Crie um repositório no GitHub e envie seu projeto local (git push)
  4. Clone seu próprio repositório em outra pasta e faça uma alteração + push
  5. Crie um arquivo README.md com Markdown básico e commite
  6. Adicione um .gitignore adequado para a linguagem que você usa
Dica de ouro: Faça commits pequenos e frequentes. Um commit = uma ideia ou mudança lógica.
